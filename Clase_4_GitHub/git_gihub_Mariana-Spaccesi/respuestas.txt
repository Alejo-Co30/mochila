1. ¿Quién inventó el sistema de control de versión Git y por qué?

Git fue creado por Linus Torvalds en 2005, el creador del kernel Linux, debido a que necesitaba una herramienta 
que pudiera gestionar mejor el flujo de trabajo de la enorme comunidad creciente que contribuía al código 
de Linux.  El origen de Git esta vinculado a ser un motor de bajo nivel sobre el cual otros pudieran escibir la interfaz del usuario.
Originalmente, la comunidad de desarrolladores de Linux utilizaba BitKeeper, un sistema de control de versiones distribuido y comercial. Por 
ser un software comercial, Linus se propuso crear su propio sistema de control de vresiones.

Linus Torvalds creó esta herramienta como medio de gestionar parches y colaboraciones en el código del kernel 
Linux. Resulta curioso que el mismo se refirio a git primeramente como una herramienta de distribucion y archivado de ficheros,
no como un sistema de control de versiones. A los dias de tener creada la herramienta, liberó el código que había creado como software libre .
Actualmente, Junio Hamano se encarga del mantenimiento del software de Git junto a 280 programadores.

3. ¿Hay otra forma que no sea la terminal para trabajar con GitHub?
Se puede trabajar directo desde la página web subiendo los archivos o mediante la aplicación desktop.
La desventaja de ambos es que no son formas optimas de trabajar.
Esto se debe a que conlleva mucho mas tiempo interactuar con la interfaz grafica que a traves de la CLI.
